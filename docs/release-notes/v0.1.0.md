## SAGE-AGENT v0.1.0

**An offline-first, thick-client mobile AI companion. The device is the agent; the backend extends its reach.**

This is the first tagged release. It completes the full build (Phases 0–6): the
device owns the ReAct loop, routing, voice, code execution, memory, and OS
integration; the backend is a stateless proxy. All platform-agnostic
orchestration logic is unit-tested with no device; the native bindings ship as
real Swift/Kotlin with documented on-device validation procedures.

### Highlights

- **Arbiter Core** — on-device `ArbiterRouter` (5 signals → model + local/cloud target), `ReActLoop` (owns the loop; uniform local/cloud inference target; graceful degradation), and `ToolDomainRouter` (authoritative two-domain dispatch).
- **Voice loop** — platform-agnostic `VoicePipeline` state machine (wake → capture → STT → think → TTS) with an injectable latency tracker; device engines: Porcupine wake word, whisper.rn STT, and a native Piper (ONNX) `SageTts` module. Runs fully offline.
- **Code sandbox** — `JsSandbox` contract with a CI-verified WASM QuickJS runner and a `SandboxManager` unifying local QuickJS (`execute_js`, `render_prototype`) and cloud E2B (`execute_python`). Proven isolation.
- **Search & memory** — offline embedder + vector store + lifecycle + opaque memory injection (`memory-core`); device sqlite-vec store via op-sqlite + SQLCipher; server-side Tavily→Jina deep research.
- **Deep OS integrations** — native Contacts, Calendar, Reminders, and sandboxed File System via the `SageOs` module, with graceful `PERMISSION_DENIED` handling.
- **SAGE Backend v3** — stateless Express proxy with a pluggable OpenAI-compatible upstream adapter (mock default; OpenRouter / Azure AI Foundry), model allowlist, and a cloud tool runtime.

### Constitutional constraints (binding)

1. The device owns the ReAct loop — the server runs exactly one inference cycle per request.
2. The `ArbiterRouter` is fully on-device — the server does an allowlist check, not routing.
3. `POST /api/sage/infer` is a stateless proxy: validate, forward, stream, return.
4. The two-domain tool registry is authoritative — mobile tools never run on the server; cloud tools fail when offline.
5. Memory is on-device and never synced — the backend treats `memories[]` as opaque prompt text.
6. No new features on deprecated paths.
